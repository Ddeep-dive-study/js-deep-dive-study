**Ajax는 JavaScript와 XML을 이용한 비동기적 정보 교환 기법!**

브라우저가 서버에게 비동기 방식으로 데이터를 요청하고, 서버가 응답한 데이터를 수신하여 웹페이지를 동적으로 갱신하는 프로그래밍 방식

- XMLHttpRequest 객체 기반으로 동작

### **XMLHttpRequest**

<aside>
💡

XMLHttpRequest 객체란?
- 브라우저에서 제공하는 WEB API로, HTTP 비동기 통신을 위한 메서드와 프로퍼티를 제공

- 브라우저 환경에서만 사용 가능

</aside>

**HTTP 요청**

```jsx
// 1. XMLHttpRequest 생성자 함수 호출하여 생성
const xhr = new XMLHttpRequest();

// 2. HTTP 요청 초기화 
xhr.open('GET', '/users');

// 3. 요청 헤더 설정
xhr.setRequestHeader('content-type', 'application/json');

// 4. HTTP 요청 전송 -> 서버 : 응답 반환 
xhr.send();

// load 이벤트는 HTTP 요청이 성공적으로 완료된 경우 발생한다. 
xhr.onload = () => { 
	if (xhr.status === 200) {
		console.log(JSON.parse(xhr.response)); // {userId: 1, id: 1, title: "delectus aut autem", completed: false}
	} 
	else {
		console.error('Error', xhr.status, xhr.statusText);
	} 
};
```

## 기존 렌더링 방식 🧐

완전한 **HTML 전체를** 서버로부터 전송받아 웹페이지 전체를 처음부터 다시 렌더링하는 방식

- 화면이 전환되면 서버로부터 새로운 HTML을 받아서 전체를 다시 렌더링
- **즉!!! html 파싱하고 js 파싱해서 돔트리랑 렌더트리를 처음부터 생성해야 한다는 말**

문제점

1. 변경할 필요가 없는 부분을 포함한 HTML 전체를 받아옴 → 불필요한 데이터 통신
2. 렌더링 시 화면이 깜박이는 현상 발생
3. 클라이언트-서버가 동기 방식으로 통신하기 때문에 서버로부터 응답이 있을 때까지 다음 태스크는 블로킹됨

## Ajax의 등장 🍀

서버로부터 웹페이지의 변경에 필요한 **데이터(JSON, XML)**만 비동기 방식으로 전송받아서 필요한 부분만 한정적으로 렌더링

- 기존의 DOM 트리는 그대로 유지되고, JavaScript로 특정 부분의 DOM만 조작해서 변경된 부분 업데이트 → 렌더트리 재계산

장점

1. 변경할 부분을 갱신하는데 필요한 데이터만 서버로부터 전송받기 때문에 불필요한 데이터 통신 발생 X
2. 렌더링 시 화면이 깜박이는 현상 X
3. 클라이언트-서버가 비동기 방식으로 통신하기 때문에 서버에게 요청을 보낸 이후 블로킹 X
